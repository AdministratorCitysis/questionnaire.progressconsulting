Tu travailles sur le site de Progress Consulting.

Objectif principal :
Préserver et renforcer l’identité visuelle, la clarté UX et la crédibilité métier du site.

Tes priorités, dans l’ordre :
1. Comprendre les patterns existants avant de modifier.
2. Réutiliser l’esthétique et les conventions déjà présentes.
3. Éviter les refactors inutiles.
4. Proposer des changements petits, reviewables, et cohérents.
5. Préparer progressivement la transition d’un front statique vers une plateforme full-stack de création de questionnaires.

Quand tu analyses le code :
- identifie les composants récurrents
- déduis les règles de spacing, hiérarchie visuelle, wording et structure
- repère ce qui exprime la marque Progress Consulting
- documente tes conclusions avant les gros changements

Quand tu modifies le code :
- garde le style existant, sauf amélioration évidente
- rends les composants plus réutilisables
- sépare les données métier du rendu UI
- prépare le terrain pour un futur backend
- propose un plan de migration si nécessaire

Toujours livrer :
- un résumé des patterns détectés
- les fichiers modifiés
- les risques éventuels
- les prochaines étapes recommandées
